# 食譜收藏夾 PRD 文件

## 1. **專案概述**

### 背景與動機
在資訊爆炸的時代，人們常常可以在社交媒體、論壇或網頁上看到各式各樣的食譜，但缺乏一個統整的工具來記錄、保存並快速尋找過去收藏過的菜色。為了幫助使用者更有效率地管理他們喜愛的食譜，我們決定開發「食譜收藏夾」系統。

### 目標用戶
- **一般人（使用者）**：想要尋找、儲存與整理自身食譜清單的料理愛好者或自煮族群。
- **管理員**：負責管理平台系統，確保內容品質與管理平台運作。

### 核心價值主張
提供一個快速、直觀且易於管理的食譜收藏平台，解決使用者收藏食譜散落在各處、難以整理且找不到的痛點。

---

## 2. **功能需求**

系統包含以下 5 項主要功能：

1. **儲存食譜**
   - 作為 **一般人**，我希望 **可以新增與儲存這道料理的名稱、食材細節與步驟等資訊**，以便 **未來需要時可以隨時查閱及製作**。
2. **瀏覽食譜**
   - 作為 **一般人**，我希望 **可以瀏覽所收藏的食譜清單與詳細介紹頁面**，以便 **決定今天想做的晚餐或預覽食材清單**。
3. **搜尋食譜**
   - 作為 **一般人**，我希望 **可以透過名稱或關鍵字搜尋我的食譜庫**，以便 **在大量收藏中快速找到我想煮的那道菜**。
4. **分類與標籤管理**
   - 作為 **一般人**，我希望 **能為食譜建立分類或加上自訂標籤（例如：中式、甜點、減脂、快速上菜）**，以便 **有系統地整理並根據當下情境篩選食譜**。
5. **使用者與系統管理**
   - 作為 **管理員**，我希望 **可以登入並檢視系統狀況、管理使用者與移除不當的內容**，以便 **維持平台的安全性與良好使用體驗**。
   - 作為 **一般人**，我希望 **可以註冊或登入我的個人帳號**，以便 **我的食譜資料能不受裝置限制，安全地儲存在自己的帳戶中**。

---

## 3. **非功能需求**

- **技術限制**：
  - 後端系統主要依賴 Python 的 **Flask** 框架。
  - 前端渲染使用 **Jinja2** 模板。
  - 資料庫使用 **SQLite** 進行關聯式資料儲存。
- **效能與安全考量**：
  - 回應時間：確保基本的搜尋與瀏覽頁面能在 2 秒內完成載入。
  - 資訊安全：防止基本的 SQL Injection 與 XSS 攻擊；使用者密碼必須採用 bcrypt 進行加密儲存；需要有防護措施避免未授權者存取他人筆記與食譜。

---

## 4. **MVP 範圍 (最小可行性產品)**

- **Must Have（必須具備，核心）**：
  - 使用者註冊、登入與登出流程。
  - 儲存食譜（包含基本欄位：食譜名稱、食材、步驟、圖片連結）。
  - 首頁瀏覽食譜列表與進入觀看單一食譜詳細頁面。
  - 關鍵字搜尋食譜。
- **Should Have（應該具備，重要選項）**：
  - 食譜的分類與自訂標籤功能。
  - 修改與刪除自身建立的食譜。
- **Nice to Have（有會更好，加分項目）**：
  - 系統後台與管理員權限控制。
  - 為食譜寫下實作後的心得備註留言。
  - 匯入或從外部網站自動擷取食譜資訊功能。
